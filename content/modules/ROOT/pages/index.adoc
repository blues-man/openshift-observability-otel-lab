= Full-Stack Observability on OpenShift with OpenTelemetry

A hands-on guide to strategic sampling and data optimization.

Based on the KubeCon EU 2026 workshop by Pavol Loffay (Red Hat).

[cols="1h,3"]
|===
| Duration | ~90 minutes
| Audience | Intermediate developers and SREs familiar with OpenShift basics
|===

== Overview

In this workshop you will deploy OpenShift's native observability stack and learn practical techniques for controlling telemetry volume without losing visibility.

== What you will learn

* Deploy the platform observability stack -- Tempo, Loki, and the Cluster Observability Operator (COO)
* Profile telemetry data to understand baseline volume and cost
* Implement head-based sampling to reduce trace ingestion at the source
* Implement tail-based sampling to keep only the traces that matter
* Reduce log volume with deduplication

== Prerequisites

* Access to an OpenShift {ocp_version} cluster (provided)
* Basic Kubernetes and OpenShift knowledge
* Familiarity with observability concepts (traces, metrics, logs)
